# 11. 스프린트 1 실행 백로그 (2주)

## 스프린트 목표
플랫폼 기반을 안정화하고, 문의/업로드 최소 기능을 데모 가능한 상태로 만든다.

## 포함 범위 (P0)
- 에픽 A: 플랫폼 기반 (핵심)
- 에픽 B: 문의/업로드 최소 기능

---

## 백로그 목록

## S1-01 저장소/코딩 규약 확정
- 우선순위: P0
- 담당: FE Lead, BE Lead
- 작업:
  - FE lint/prettier/tsconfig strict 고정
  - BE 패키지 규약 문서화
  - PR 템플릿/커밋 컨벤션 정의
- 완료기준:
  - 규약 위반 시 CI 실패

## S1-02 CI 파이프라인 1차
- 우선순위: P0
- 담당: DevOps, BE
- 작업:
  - FE build 잡
  - BE gradle build 잡
  - 브랜치 보호 규칙 적용
- 완료기준:
  - PR에서 자동 검증 통과 시만 머지 가능

## S1-03 문의 등록 API + 기본 화면
- 우선순위: P0
- 담당: BE, FE
- 작업:
  - 문의 생성 API
  - 프론트 문의 입력 폼
  - 기본 유효성 검사
- 완료기준:
  - 문의 생성→조회 E2E 성공

## S1-04 파일 업로드 API + UI
- 우선순위: P0
- 담당: BE, FE
- 작업:
  - multipart 업로드 API
  - 업로드 진행률 UI
  - 실패 재시도 버튼
- 완료기준:
  - PDF/Word 파일 업로드 성공

## S1-05 DB 기초 스키마 + 마이그레이션
- 우선순위: P0
- 담당: BE
- 작업:
  - inquiry/document/document_version 테이블
  - 마이그레이션 스크립트 도입
- 완료기준:
  - 로컬/CI에서 migration 적용 성공

## S1-06 운영 로그 최소셋
- 우선순위: P1
- 담당: BE
- 작업:
  - 요청 ID, 문의 ID, 업로드 결과 로그
  - 에러 코드 표준화
- 완료기준:
  - 문의/업로드 실패 원인 추적 가능
