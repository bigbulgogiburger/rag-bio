# 05. 에픽 D — 커뮤니케이션/승인플로우 (P1)

## 목표
고객 커뮤니케이션을 자동 초안화하되, 승인 기반으로 안전하게 발송한다.

## Story D-1: 이메일 초안 생성
### FE
- 제목/본문/첨부 안내 편집 UI
### BE
- 템플릿 렌더링(톤 선택)
### DB
- outbound_message(draft) 저장
### AC
- CS가 즉시 수정 가능한 품질의 초안 생성

## Story D-2: 고객 메시지 초안 생성
### FE
- 짧은 버전/상세 버전 탭 제공
### BE
- 채널 길이 제한 반영 생성
### DB
- message variant 저장
### AC
- 최소 2개 변형 텍스트 제공

## Story D-3: 승인 워크플로우
### FE
- Draft→Review→Approved→Sent 상태 UI
### BE
- 권한 체크 + 상태머신 전이
### DB
- approval/approval_history
### AC
- 승인 없는 발송 불가

## Story D-4: 발송 어댑터/복구
### FE
- 발송 실패 알림/재시도 버튼
### BE
- provider adapter + retry + idempotency key
### DB
- delivery_attempt 로그
### AC
- 중복 발송 방지

## 감사/컴플라이언스
- 누가/언제/무엇을 승인·발송했는지 추적 가능
