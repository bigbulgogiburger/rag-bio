# 12. 스프린트 전체 백로그 (배포 시점까지)

기준: 2주 스프린트, 총 6개 스프린트(MVP 출시)

---

## Sprint 1 — 플랫폼 기반 + 문의/업로드 최소 기능
### 목표
기본 개발체계와 문의/업로드 데모 가능 상태 확보

### 백로그
- S1-01 저장소/코딩 규약 확정 (P0)
- S1-02 CI 파이프라인 1차 구축 (P0)
- S1-03 문의 등록 API + 화면 (P0)
- S1-04 파일 업로드 API + 진행률 UI (P0)
- S1-05 DB 기초 스키마 + 마이그레이션 (P0)
- S1-06 운영 로그 최소셋 (P1)

### 스프린트 종료 기준
- 문의 등록/조회 + PDF/Word 업로드가 데모 가능

---

## Sprint 2 — OCR/청킹/임베딩/인덱싱
### 목표
문서가 검색 가능한 지식 단위로 안정 변환되도록 구축

### 백로그
- S2-01 문서 파싱 파이프라인(PDF/Word) (P0)
- S2-02 OCR 분기 처리 + confidence 저장 (P0)
- S2-03 청킹 정책 적용(chunk=1000, overlap) (P0)
- S2-04 OpenAI 임베딩 + Vector Store 업서트 (P0)
- S2-05 인덱싱 상태 조회 API + UI (P0)
- S2-06 실패 재처리 API/버튼 (P1)
- S2-07 인덱싱 이벤트 로그/메트릭 (P1)

### 스프린트 종료 기준
- 텍스트형/스캔형 PDF 및 Word가 인덱싱 완료 상태까지 도달

---

## Sprint 3 — RAG 검색 + 기술 검증(verdict)
### 목표
근거 기반 검색 결과와 맞/틀/조건부 판정 제공

### 백로그
- S3-01 질의 전처리 + 검색 API(top-k/threshold) (P0)
- S3-02 retrieval evidence 저장/조회 (P0)
- S3-03 검증 규칙 엔진(verdict policy) (P0)
- S3-04 상충 근거 감지 + risk flag (P0)
- S3-05 verdict 결과 UI(배지/신뢰도/리스크) (P0)
- S3-06 품질 평가셋 v1 구축 (P1)
- S3-07 정확도/출처포함률 리포트 자동화 초안 (P1)

### 스프린트 종료 기준
- 단일 질의에 대해 verdict + 근거 목록이 재현 가능하게 제공

---

## Sprint 4 — 답변 생성 + 출처 표기 + 멀티에이전트 체인
### 목표
Retriever→Verifier→Composer 체인으로 최종 답변 생성

### 백로그
- S4-01 답변 생성 서비스(근거 인용 강제) (P0)
- S4-02 structured output schema 적용 (P0)
- S4-03 출처 UI(문서/페이지/chunk 하이라이트) (P0)
- S4-04 멀티에이전트 오케스트레이션 체인화 (P0)
- S4-05 단계별 agent run 로그 저장 (P0)
- S4-06 실패 단계 폴백 처리 (P1)
- S4-07 답변 품질 회귀 테스트 세트 추가 (P1)

### 스프린트 종료 기준
- end-to-end: 질문 → 검색 → 검증 → 답변이 자동 실행

---

## Sprint 5 — 커뮤니케이션 초안 + 승인/발송 + 보안/운영
### 목표
초안 생성과 승인 기반 발송을 안전하게 운영 가능한 상태로 제공

### 백로그
- S5-01 이메일 초안 생성(템플릿/톤) (P0)
- S5-02 고객 메시지 초안(짧은/상세 버전) (P0)
- S5-03 승인 상태머신(Draft→Review→Approved→Sent) (P0)
- S5-04 승인 권한 체크(RBAC) (P0)
- S5-05 발송 어댑터 + 재시도 + idempotency (P0)
- S5-06 승인/발송 감사로그 조회 (P1)
- S5-07 운영 대시보드/알람 1차 (P1)
- S5-08 민감정보 마스킹 로깅 (P1)

### 스프린트 종료 기준
- 승인 없는 발송이 기술적으로 차단되고, 승인 후 정상 발송 가능

---

## Sprint 6 — 베타 운영 + 튜닝 + 릴리즈
### 목표
베타 피드백 반영 및 프로덕트 배포(Go-live)

### 백로그
- S6-01 파일럿 사용자 그룹 운영 (P0)
- S6-02 핵심 KPI 측정(정확도/P95/재작성률) (P0)
- S6-03 OCR/RAG 파라미터 튜닝(top-k/threshold/overlap) (P0)
- S6-04 성능 병목 개선(백엔드/DB) (P0)
- S6-05 장애 대응 런북/온콜 시나리오 확정 (P1)
- S6-06 보안 점검(RBAC/권한/로그) (P1)
- S6-07 릴리즈 체크리스트 완료(99 문서 연동) (P0)
- S6-08 Go/No-Go 회의 및 배포 승인 (P0)

### 스프린트 종료 기준
- Go-live 승인 조건 충족, 운영 전환 완료

---

## 스프린트 공통 운영 룰
- 각 스토리는 AC/DoD 없으면 시작 금지
- 스프린트 중간변경은 PM 승인 + 영향도 공유 필수
- 장애/품질 이슈는 기능개발보다 우선

## 추적 지표 (매 스프린트)
- 계획 대비 완료율
- 이월 스토리 수
- 결함 발생/회귀 건수
- KPI 추세(정확도, P95, 출처 포함률)
