# 13. S1-02 CI / 브랜치 보호 설정 가이드

## 1) CI 워크플로우
- 파일: `.github/workflows/ci.yml`
- 트리거:
  - PR to `main`
  - push to `main`
- 체크 잡:
  - `Frontend Build`
  - `Backend Build & Test`

## 2) GitHub 브랜치 보호 (수동 설정)
GitHub 저장소에서 아래로 이동:
- Settings → Branches → Branch protection rules → Add rule

권장 설정:
- Branch name pattern: `main`
- ✅ Require a pull request before merging
- ✅ Require status checks to pass before merging
  - Required checks:
    - `Frontend Build`
    - `Backend Build & Test`
- ✅ Require branches to be up to date before merging
- (선택) ✅ Require approvals (최소 1명)
- (선택) ✅ Restrict who can push to matching branches

## 3) 검증 시나리오
1. 정상 PR 생성 → 두 체크 green 확인
2. 의도적 실패 커밋(PR) → red 확인
3. red 상태에서 merge 차단 확인

## 4) 완료 기준 (DoD)
- [ ] CI가 PR마다 자동 실행됨
- [ ] 필수 상태 체크 2개가 보호 규칙에 등록됨
- [ ] 실패 체크 시 main 머지 차단 확인
