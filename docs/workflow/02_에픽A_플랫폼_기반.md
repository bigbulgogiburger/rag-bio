# 02. 에픽 A — 플랫폼 기반 (P0)

## 목표
개발/운영 생산성을 확보하는 공통 기반을 구축한다.

## Story A-1: 저장소/개발표준 정립
### FE
- ESLint/TS strict/절대경로 alias 정리
- 공통 UI 폴더 규칙 정의
### BE
- 멀티모듈 의존 규칙 정리(app-api ↔ contexts)
- 패키지 구조 규약(domain/application/infrastructure/interface)
### DB
- 마이그레이션 도구 채택(Flyway)
- 스키마 네이밍 규칙 수립
### AC
- 신규 인원이 30분 내 로컬 실행 가능

## Story A-2: CI/CD 기본 파이프라인
### FE
- install/lint/build 분리
### BE
- gradle build/test 분리
### DB
- migration dry-run 잡 추가
### AC
- PR 시 자동 검증, 실패 시 머지 차단

## Story A-3: 공통 예외/에러코드 체계
### FE
- API 에러코드별 메시지 매핑
### BE
- 표준 에러 응답 스키마 도입
### DB
- 무결성 위반 오류 코드 매핑
### AC
- 주요 에러 시나리오가 일관된 코드로 반환

## Story A-4: 환경 구성(dev/stage/prod)
### FE
- 환경별 baseURL/feature flag
### BE
- profile 별 설정 분리
### DB
- 환경별 접속 정책/권한 분리
### AC
- 환경 전환 시 코드 변경 없이 배포 가능

## 리스크
- 초기에 규약 미정으로 재작업 발생

## DoD
- 빌드 파이프라인 green
- 아키텍처/규약 문서 최신화
