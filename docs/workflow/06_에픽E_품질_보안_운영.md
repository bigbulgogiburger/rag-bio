# 06. 에픽 E — 품질/보안/운영 (P1)

## 목표
서비스 신뢰성, 안전성, 운영 대응력을 확보한다.

## Story E-1: 테스트 전략 실행
### FE
- 폼/결과뷰/상태전이 테스트
### BE
- Domain/UseCase/Integration 테스트
### DB
- migration 검증 테스트
### AC
- 핵심 플로우 자동 테스트 통과

## Story E-2: 성능/부하
### FE
- 화면 체감성능 측정(web-vitals)
### BE
- P95 지연/처리량 측정, 병목 튜닝
### DB
- 슬로우쿼리 분석/인덱스 최적화
### AC
- 성능 지표 목표 또는 개선 계획 확보

## Story E-3: 보안/RBAC/마스킹
### FE
- 권한 기반 메뉴/기능 가드
### BE
- RBAC, 입력검증, rate limit
### DB
- 최소권한 계정(app/readonly/admin)
### AC
- 무권한 접근 차단 검증 완료

## Story E-4: 관측/알람/런북
### FE
- 운영 상태 대시보드
### BE
- 구조화 로그/메트릭/트레이스
### DB
- 백업/복구 리허설
### AC
- 장애 발생 시 10분 내 대응 시작 가능

## Story E-5: 베타 운영/피드백
- 파일럿 그룹 운영
- 주간 피드백 반영
- Top10 개선 백로그 도출
