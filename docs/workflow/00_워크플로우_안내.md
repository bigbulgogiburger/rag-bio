## 문서 구성

1. `01_프로그램_계획.md` — 전체 로드맵/스프린트 계획
2. `02_에픽A_플랫폼_기반.md` — 기반/아키텍처/CI
3. `03_에픽B_수집_인덱싱.md` — 문의/업로드/OCR/인덱싱
4. `04_에픽C_RAG_검증_답변.md` — RAG/판정/답변
5. `05_에픽D_커뮤니케이션_승인플로우.md` — 이메일/메시지/승인 플로우
6. `06_에픽E_품질_보안_운영.md` — 품질/보안/운영
7. `10_에픽별_FE_BE_DB_상세.md` — 에픽별 FE/BE/DB 상세 분해

## 티켓 표준 템플릿

각 문서의 스토리는 아래 형식을 따릅니다.

- Story ID
- 목적 (Why)
- 범위 (Scope)
- 세부 Task
- 수용 기준 (Acceptance Criteria)
- 완료 정의 (Definition of Done)
- 의존성 (Dependencies)

## 우선순위 원칙

- P0: MVP 출시를 막는 핵심
- P1: MVP 품질/운영 안정화
- P2: 고도화
